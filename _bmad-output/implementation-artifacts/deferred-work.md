- "Drag label N" aria-label numbering is non-contiguous and unstable [components/Editor/LabelEditOverlay.tsx:67] — the index `i+1` is taken over the FULL `points` array while hidden / un-laid-out points (`swatchOrder === null`, `!visible`) are filtered out, so visible labels get non-contiguous numbers that shift when an earlier point's visibility/order changes. ## Deferred from: code review of 3-1-style-picker-and-live-style-switching (2026-06-15)

- Switching to a larger-radius style clips a swatch ≤8px past the canvas edge and can overlap neighbours ≤16px until the next drag [components/Editor/index.tsx:391, components/Editor/EyedropperLayer.tsx:32,103] — `handleSelectStyle` only calls `setStyle` and never re-clamps `swatchOrder`, which was bounded under the *previous* radius. `minimal`(r=40)→`float`/`float_clean`/`grid`(r=48): a swatch at `swatchOrder = canvasHeight − 40` renders with r=48 → bottom at `canvasHeight + 8`, clipped by the `<canvas>` bitmap (Konva itself doesn't clip). Adjacent swatches spaced for r=40 (≥80px) overlap when re-drawn at r=48 (need ≥96px) until the next drag re-runs `placeSwatchOnEdge`. Story 3.1 line 33 explicitly pre-authorized NOT adding a re-layout here, so this is a scoped/accepted gap — but note the story's justification ("40 < 48 keeps positions in-bounds") is logically backwards and the dev Completion Note "No out-of-bounds swatch observed" missed the minimal→larger extreme case. Fix when the swatch model is next touched: re-clamp `swatchOrder` to `[r, edge−r]` (or call `assignSwatchLayout`) inside `handleSelectStyle` when `swatchRadius` changes.
- New filled `dot` marker is near-invisible on light areas [styles.json `grid` style, components/Editor/EyedropperLayer.tsx:156] — all 4 styles set `markerColor: "#ffffff"`. The `dot` branch added in 3.1 draws a 6px white filled circle with no stroke; on light regions of a drawing it can disappear (the hollow ring stays more legible against its surroundings). AC-compliant (renders the style's own `markerColor`); the remedy lives in `styles.json` (a darker `markerColor` or a thin contrasting stroke on dots), which was out of scope for 3.1. Revisit in a future style-tuning story.

## Deferred from: code review of 4-1-jpeg-export-and-download (2026-06-29)

- AC7 capture uses a fixed two-`requestAnimationFrame` wait, not synchronized to Konva draw completion or web-font readiness [components/Editor/index.tsx:513-520] — leaving edit mode then waiting two frames (React commit + one paint) is the story-prescribed recipe, but Konva batch draws aren't tied to RAF count and the italic label font (Cormorant Garamond) may not be loaded on the first paint after the LabelLayer mounts. Rarely, the exported JPEG could show missing/fallback-font labels. Refinement: `await document.fonts.ready` and/or an explicit `layer.batchDraw()` before `toDataURL`. Low likelihood (export is usually not the first action after labels appear), so deferred rather than patched.
- Very large uploads exceed the browser canvas size cap at 2× pixelRatio [components/Editor/index.tsx:507,520] — `pixelRatio = 2·canvasWidth/displayWidth` targets 2× the original image width, so a very large upload (e.g. 6000px wide → ~12000×21333 export) can exceed the browser's max canvas area; `toDataURL` then returns `""`, the client POSTs `data:,`, the server 400s, and the export silently fails (button just re-enables, no error UI per AC6). Root cause is pre-existing: `/api/upload` does not clamp source dimensions. Fix belongs with an upload-side max-dimension clamp (or a client-side pixelRatio cap + user feedback), out of scope for 4.1.
- `handleExport` has no automated test [components/Editor/index.tsx:498-529] — the two most behavior-dense, highest-risk pieces of the story (the AC4 pixelRatio computation and the AC7 edit-mode toggle/2-frame wait) are verified only by code inspection. The story explicitly sanctions skipping a full EditorShell render (3.3/3.4 precedent), so this is compliant, but AC4 dimensions and AC7 labels remain runtime-unverified. Suggestion: extract the pure pixelRatio math into a `lib/` helper (mirrors the `lib/download.ts` "extract the hard-to-test bit" pattern) so AC4 becomes unit-testable without standing up the shell.
